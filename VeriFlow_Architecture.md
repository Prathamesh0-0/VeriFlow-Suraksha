# VeriFlow — Document Forensics Engine Architecture

VeriFlow is a real-time, fully offline document forensic engine designed to detect tampered financial documents (salary slips, bank statements, loan applications, and tax returns) submitted during underwriting processes. 

It uses a deterministic, two-tiered pipeline to analyze documents both at the structural level (pixel/metadata) and the semantic level (cross-document financial coherence).

## 1. System Architecture & Tech Stack

VeriFlow is built as a decoupled Full-Stack application prioritizing speed, local execution (privacy), and rigorous deterministic logic.

### Technical Stack
*   **Frontend**: React (Vite), plain CSS (Static Enterprise Bank UI design).
*   **Backend**: FastAPI (Python), providing asynchronous HTTP endpoints and background task execution.
*   **Forensic Engine Core**:
    *   **PyMuPDF (fitz)**: For extracting native PDF text layers, fonts, sizes, and internal metadata.
    *   **EasyOCR / OpenCV / NumPy**: For image-based OCR extraction, Error Level Analysis (ELA) heatmap generation, and image manipulation.
*   **Local Processing**: 100% offline. No external APIs (like OpenAI) are used, ensuring PII (Personally Identifiable Information) never leaves the server.

---

## 2. Execution Flow

When a packet of documents is uploaded, VeriFlow processes them through a sequential, multi-layered pipeline:

1.  **Ingestion & Classification**: Documents are uploaded via the frontend. The backend detects the document type (e.g., Salary Slip, Bank Statement, ITR) based on content patterns and filename heuristics.
2.  **Layer 1: Structural DNA Analysis**: Every document is individually analyzed for low-level visual and metadata tampering.
3.  **Layer 2: Coherence Engine**: Text and financial fields are extracted across all documents and cross-referenced to ensure logical consistency.
4.  **Forensic Intelligence Engine**: A 12-rule deterministic engine runs against the extracted data to flag financial anomalies (e.g., impossible math, income inflation).
5.  **Aggregation**: A final weighted Risk Score (0-100) is calculated, generating a detailed report with a verdict (`CLEAN`, `SUSPICIOUS`, or `TAMPERED`).

## 3. Pipeline Deep Dive

### Layer 1: Structural DNA (Document Level)
This layer catches "Packet A" fraud (visually manipulated documents) before reading any text.

*   **Syntax & Geometry Analysis (`syntax_geometry.py`)**: 
    *   Parses the PDF content stream.
    *   Checks for subtle font deviations (e.g., if a document uses Arial 10pt everywhere except for the net pay figure which is 10.5pt).
    *   Checks for baseline drift (misaligned Y-coordinates indicating pasted text).
*   **Chronological Forensics (`chronological.py`)**: 
    *   Reads PDF XMP metadata.
    *   Validates creation vs. modification timestamps.
    *   Flags documents generated by known PDF editors (e.g., Adobe Acrobat, iLovePDF).
*   **Error Level Analysis / ELA (`ela.py`)**:
    *   Resaves image pages at a known compression rate.
    *   Generates a heatmap highlighting areas with different compression levels, which typically indicates a spliced or photoshopped region.

### Layer 2: Coherence Engine (Cross-Document Level)
This layer catches "Packet B, C, & D" fraud (forged numbers, income inflation) by treating the documents as a connected graph of financial data.

*   **OCR & Field Extraction (`ocr_extractor.py`)**: 
    *   Extracts specific fields based on the classified document type using regex patterns and proximity fallbacks.
*   **Cross-Document Validation (`cross_document.py`)**:
    *   Validates that the `Net Pay` on a Salary Slip perfectly matches the `Salary Credit` on the Bank Statement.
    *   Checks that the `Applicant Name` and `Employer Name` are consistent across all files.
    *   Verifies that the `Monthly Income` claimed on a Loan Application matches the actual Salary Slip.
*   **Tax Logic Engine (`tax_logic.py`)**:
    *   Reverse-engineers the Indian Tax System (FY 24-25 New Tax Regime).
    *   Takes the stated Gross Salary and recalculates the expected TDS (Income Tax) and EPF (Provident Fund).
    *   If the forged salary doesn't match the required mathematical tax deductions, it flags the document as fabricated.

### The Forensic Intelligence Engine (`local_ai.py`)
A hardcoded, 12-rule deterministic system that replaces the need for an LLM. It instantly evaluates all extracted data against rules such as:
1.  *Gross - Deductions = Net Pay* (Arithmetic consistency).
2.  *Net Pay > Gross Pay* (Impossible math).
3.  *Suspiciously Round Numbers* (e.g., exactly ₹1,00,000).
4.  *High Debt-to-Income Ratio*.
5.  *Missing Mandatory Deductions* (High salary but ₹0 tax).
